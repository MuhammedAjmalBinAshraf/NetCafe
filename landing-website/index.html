System Requirements

Minimum requirements for running the Admin Server and Client Agent on your network.

Windows
  • ✓ Windows 10 / 11 (64-bit)
  • ✓ 4 GB RAM minimum (8 GB recommended)
  • ✓ 200 MB free disk space
  • ✓ LAN / Wi-Fi network connection
  • ⚠ Admin must run Server as Administrator
  • ⚠ Agent must run as Administrator (hosts file access)
Ubuntu / Debian Linux
  • ✓ Ubuntu 20.04+ or Debian 11+ (64-bit only)
  • ✓ 4 GB RAM minimum (8 GB recommended)
  • ✓ 200 MB free disk space
  • ✓ LAN / Wi-Fi network connection
  • ⚠ sudo access required for Client Agent
  • ⚠ iproute2 package for bandwidth limiting (tc)
  • ⚠ Agent uses pkexec (polkit) for privilege elevation at launch

Setup & Installation Guide

Follow these quick steps to deploy NetCafe Manager on your local area network (LAN).

  1. Download & Install

    Download the Server installer (.exe) and run the setup on the main operator/administrator PC.

  2. Start the Server

    Launch the NetCafe Server. Upon launch, sign in with the default operator credentials:

    Username: admin
    Password: admin
  3. Find your LAN IP

    Make sure the operator PC has a static local IP on the network (e.g. 192.168.1.10). You can verify it by running ipconfig in Command Prompt.

  1. Download & Deploy

    Download the Client Agent installer and run it on all user terminals. The agent will run fullscreen, locking the interface.

  2. Configure Connection Path

    Find the config.json path displayed on the lock screen (e.g. AppData/Roaming/agent/config.json) and set it to point to the Server PC's IP address:

    {
  "serverUrl": "ws://192.168.1.10:9000",
  "machineId": "PC-01"
}
  3. Restart Terminal

    Relaunch the client agent. It will establish a WebSocket handshake, register itself as available on the operator dashboard, and lock down user interactions until opened.
